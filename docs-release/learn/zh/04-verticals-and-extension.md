# 扩展模型：场景合同、检查器、里程碑、档案

[三原语内核](01-three-primitive-kernel.md)对你的业务领域一无所知。它只有 decision、task、fact,以及它们之间的关系——这是刻意为之的设计。让系统对*编码*、*研究*或*运维*真正有用的一切,都放在上面一层,叫做**场景合同(vertical)**。这一页讲这一层是怎么运作的,以及你要如何为自己的领域搭一个。

## 内核保持领域无关,场景合同承载业务

一个场景合同是一份**场景协议(scenario contract)**。比如`software/coding`场景合同,声明了编码项目关心哪些实体种类、它们具体化成什么样的文档脚手架、用什么检查器(checker)规则守护它们,以及允许连接它们的关系类型白名单。场景合同是一份声明式工件——一个`vertical.json`加它的 schema——不是编译出来的代码。引擎本质上是一个声明解析器:你描述业务领域的形状,它就把实体具体化出来、把关系接好、并把规则强制执行下去。

这条边界带来的收益很直接:你可以加一百个领域概念,内核代码一行都不用动。里程碑(milestone)是编码领域的概念;研究项目可能用*阶段(phases)*来代替。这两个词,内核永远都不需要认得。

## 约定优于声明

扩展系统最朴素的做法,是让你把所有东西都声明一遍。这条路的尽头是一片配置沼泽:每个新实体都要重复同样的模板代码。走向另一个极端——什么都靠推断,什么都不声明——则会陷入一座魔法迷宫,系统猜错了,你还查不出原因。

稳定的平衡点,是一个两层拆分,逐字段判断,只问一个问题:**引擎能不能从已有文件里推断出这个?**

- **能 → 交给约定。** 目录结构、文件是否存在、命名位置、schema 是否合法、一段关系的两端是否对得上——这些引擎自己扫描就能确认,结构不合法就 fail-closed。你什么都不用写。
- **不能 → 最小化声明。** 有些东西纯粹是意图,文件系统永远揭示不出来:一个实体是否*承重(load-bearing)*(违反它会不会让某道门变红?)、适用哪个检查器(checker)档案、版本兼容范围、某个语言缺失时该怎么降级。这些必须由你声明——但也仅限于这些,而且只需要几行。

结构靠检测,意图靠声明。规则就这么多。

## 三种实体,三种不同的通用扩展方式

场景合同并不是用同一种方式扩展每个实体。一共有三种形状:

- **生命周期实体(lifecycle entities)**(task、decision)享受完整待遇:一个实体种类、一份文档包脚手架、一个状态机绑定、一个承重标志、外加一个检查器档案。它们具体化成完整的文档包,其中的前置元数据(front-matter)是权威来源。
- **Schema 实体(schema entities)**(fact)只拿到*字段 schema*——一条编码发现声明`severity`/`evidence`,一条研究观察声明`source`/`confidence`。fact 刻意*没有*文档模板:它只是做任务过程中顺手记下的一条观察,所以场景合同约束的是它的字段,而不是它的形状。
- **复合实体(composite entities)**(milestone)**完全不需要新存储**。milestone 不是一个新文件夹——它是一个边界决策,加上 N 个验收决策,加上一组 task,再加上把它们组织起来的关系。它的结构靠检测;它的组成语义靠声明。

## 检查器、里程碑和档案

一个**检查器(checker)**是场景合同声明的一套承重不变式规则。你调用它的某个档案(`ha check --profile ...`),这些规则按 [fail-closed](03-gates-and-fail-closed.md) 方式运行:一旦违反某条承重不变式,写操作会被拒绝,而不是被放行。

如前所述,**里程碑(milestone)**是一个复合体——现有原语的组装,而不是第四个原语。它需要新存储吗?不需要,因为关系已经足够让你把手上的实体分组、组织起来。

**档案(dossier)**是里程碑关闭时产生的东西:一份记录*构建了什么*的文档。里程碑退出时,场景合同安装的检查器会 fail-closed,除非档案已经存在——正是这一点迫使档案被创建出来。档案(构建过程的前向记录)与全景(panorama)(向后回望的叙事性复盘)并存;两者从相反的方向,共同校准同一个里程碑。

## 搭建你自己的场景合同

因为引擎本质是个解析器,而这套拆分把声明压到了最少,用户完全可以在代码库之外编写一个新场景合同:针对场景合同和预设协议写好 JSON,引擎就会把它具体化出来。约定层让这件事*安全*——结构写错了就"接不上",你得到的是一个明确的错误,而不是悄无声息的数据腐坏——意图层则把负担控制在几行之内。一个陌生人能在不碰内核代码的情况下,搭起一个最小可运行的场景合同,这不是副作用,这就是设计初衷。
